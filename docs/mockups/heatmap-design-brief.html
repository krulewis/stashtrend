Layout Spec

The grid fits in 375px (minimum) to 390px (standard) viewports with no horizontal scrolling. Row labels take 127px, an 8px gap separates labels from the dot zone, and the remaining space distributes evenly across 6 columns.

Interaction

Tap group header to expand/collapse. Chevron rotates 90deg. Content animates with grid-template-rows: 0fr/1fr (300ms, --ease-smooth). All groups default to collapsed.

Window picker selects the 6-month range. Tapping opens a calendar-style month grid; selecting a month sets the start of the window. Follows the same ARIA combobox/listbox pattern as MonthDropdown.

Dots are read-only — no tap, hover, or focus interaction.

Zone Logic

Reuses getBudgetZone(actual, budgeted) from utils/budgetUtils.js with the existing WARNING_THRESHOLD = 0.85. Aggregate dots use group-level totals (sum of actual, sum of budgeted).

Accessibility

Grid uses role="grid" with role="row" / role="gridcell". Each dot cell has an aria-label describing category, month, percentage, and zone status. Group headers use aria-expanded and aria-controls. Picker follows combobox pattern with keyboard nav (Arrow/Enter/Escape). prefers-reduced-motion disables all transitions.

Integration

Third pane (index 2) in HorizontalSwipeContainer. View indicator shows 3 dots. Data lifted to BudgetPage — heatmap pane receives pre-fetched multi-month data as props. Reuses existing budget_custom_groups ordering.
