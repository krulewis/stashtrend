Milestone Progress — 8 Visualization Concepts

Interactive mockups for evaluating how investable capital milestones should be displayed on the Net Worth page. Each concept uses the same sample data.

Sample data: Half-Mil $500K achieved  ·  First Million $1M achieved  ·  Fat FIRE $2M at 62%  ·  Nest Egg $1.7M target at 73%   |   Current: $1.24M investable capital

Concept 1

Stack of Flags — Milestone Card Grid

A responsive card grid — one card per milestone. Achieved milestones display a sealed green state with a checkmark and date crossed. In-progress milestones show a horizontal progress bar with percentage and projected date. The nest egg target card has a cobalt border glow.

Strengths

  • Directly answers all 5 questions: current, progress, projected date, on-pace, achieved vs future
  • Mobile trivially handled — cards stack to single column

Concept 2

Summit Climb — Vertical Milestone Track

A vertical track with milestone nodes in order. Achieved milestones have filled green circles with checkmarks above the current position. A glowing cobalt dot marks "you are here." Future milestones sit below with hollow circles and projected years.

Strengths

  • Strong "journey" metaphor — past achievements are above, goals are ahead
  • Achieved vs future distinction is spatially obvious at a glance
  • Feels like a leveling system / achievement tree, which is engaging
  • The glowing "current position dot" is distinctive versus any current Stashtrend pattern

Weaknesses

  • Non-proportional spacing loses financial distance info — $500K to $1M looks same as $1M to $2M
  • Proportional spacing creates usability issues: nodes near each other collapse
  • Does not answer "am I on pace?" quantitatively
  • With 1-2 milestones, the track is short and feels sparse

Concept 3

Fuel Gauge — Radial Arc per Milestone

One SVG speedometer-style arc gauge per milestone. Arc fill color transitions by state: green when achieved, cobalt when in progress, amber when further off. Center label shows current value. Horizontal scroll row on desktop, vertical stack on mobile.

Strengths

  • Visually striking — no other component in Stashtrend uses radial SVG
  • Percentage readout is instantly legible at center of arc
  • Familiar gauge metaphor transfers from real-world mental models
  • Multiple gauges in a row are visually distinctive and scannable

  • Very small percentages (under 3%) look like no progress at all
  • Achieved state is less natural — a "full" arc looks same at 100% or 200%
  • Does not show projected date trajectory or history visually

Concept 4

Mountain Skyline — Investable Capital History + Projection

A standalone area chart showing investable capital over time. Left half is historical (cobalt area fill), right half is a dashed projection line to retirement. Milestone amounts appear as horizontal reference lines — green for achieved, amber for future. A vertical "today" line separates history from projection.

Strengths

  • Shows actual growth trajectory — users see acceleration or stagnation
  • Projection line makes "when will I get there?" visually intuitive
  • Historical achievement dates visible as chart crosses each milestone line
  • Closest to what best-in-class financial apps (Empower, Wealthfront) show
  • mergeHistoryWithProjection() already exists and handles the data merge

Weaknesses

  • Recharts ReferenceLine label collision problems are severe with multiple close milestones
  • Adding a third chart to the Net Worth page increases scroll length significantly
  • Y-axis domain must accommodate highest milestone, which can dwarf current progress
  • Projection requires expected_return_pct — degrades to history-only if not set

Concept 6

Runway to Liftoff — Stepped Progress Staircase

A horizontal staircase where each step represents a milestone with proportional width based on dollar gap. Current investable capital fills the stairs like rising water — green for achieved steps, cobalt for the partially-filled current step. The nest egg target sits at the top right with a plane icon.

Strengths

  • Journey metaphor is clear — you can see how far you've climbed
  • Proportional step widths communicate that some milestones are farther apart
  • The "rising water" fill metaphor is visually intuitive and satisfying
  • Compact — the entire staircase fits in a fixed-height band (~180px)

Weaknesses

  • Wildly disproportionate milestone gaps break the proportional layout
  • With many milestones (8+), step labels overlap — needs rotation or hover-only labels
  • Does not explicitly state projected dates — requires tooltip or separate label
  • Horizontal staircase becomes very cramped at 375px mobile — significant responsive work needed

Concept 7

Twin Lines — History vs Projection Dual Chart

A compact line chart with a fixed label column on the left. Milestone thresholds appear as horizontal reference bands spanning the full chart width. The label column anchors each band's label outside the SVG to eliminate Recharts label collision. Today's line separates history from projection.

Concept 8

Distance to Summit — Single Next-Milestone Hero Card

A large hero card focusing entirely on the next unachieved milestone. A CSS conic-gradient progress ring shows percentage at a glance. A pill strip below shows all milestones — achieved as green pills, in-progress as cobalt, future as ghost. Clicking a pill cycles the hero to that milestone.

Strengths

  • Maximum focus — one milestone at a time means no label clutter or cramped layout
  • Large type for percentage and amounts is easy to read on mobile and desktop
  • Interactive — click any pill to inspect any milestone without leaving the component
  • Best mobile experience of all concepts — hero layout scales to any screen width
  • The "next milestone" framing creates forward momentum — always points to the next target

Weaknesses

  • Only one milestone visible at a time — comparing all milestones requires clicking through
  • "Next milestone" focus misses the "what have I passed?" question unless the strip is noticed
  • The cycling interaction may be non-obvious to first-time users without an explicit affordance
  • If the user has only 1 milestone, the strip is a single pill — interaction pattern is pointless
